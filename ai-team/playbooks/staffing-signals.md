# Staffing Signals

This team starts with six roles on purpose. Add specialists only when a repeated bottleneck justifies more coordination overhead.

## Add A UI/UX Specialist When

- user flows repeatedly stall because specs lack interaction detail
- visual or usability rework appears in three or more consecutive tasks

## Add A Security Specialist When

- security review becomes a release blocker in consecutive releases
- sensitive data, auth, or compliance work becomes a recurring theme

## Add A Data/AI Specialist When

- the roadmap is dominated by data pipelines, evaluation, or model integration work
- core design decisions depend on dataset quality or model behavior

## Add An Independent Reviewer When

- the tech lead becomes a review bottleneck for more than a week
- architectural review quality drops because implementation pressure is too high

## Do Not Add A Role Yet When

- the problem appears in a single task only
- the current bottleneck is unclear or inconsistent
- better templates or stronger escalation would solve the issue
